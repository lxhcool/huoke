# Joinf / LinkedIn 增强版字段清单

## 1. 文档目的

本清单用于明确真实数据接入阶段，需要从 `cloud.joinf.com` 与 `LinkedIn` 获取的字段范围，作为后续采集、清洗、主档映射和搜索展示的统一依据。

当前目标不是一次性把所有字段都用于页面展示，而是先统一“采什么、怎么命名、属于哪个公司/联系人/海关对象”。

## 2. 数据源范围

本阶段纳入的外部页面数据源包括：

- `cloud.joinf.com`
  - `数据营销 -> 商业数据`
  - `数据营销 -> 海关数据`
- `LinkedIn`
  - 公司主页
  - 联系人个人主页

## 3. 字段分层原则

所有字段按以下三层理解：

- `核心字段`：首版搜索、排序、画像必须依赖的字段
- `增强字段`：有助于提高线索质量和推荐解释
- `审计字段`：用于追踪来源、抓取时间、原始页面信息

## 4. Joinf 商业数据字段

### 4.1 公司基础信息

- 公司名称
- 标准公司名称
- 公司别名
- 国家/地区
- 城市
- 公司地址
- 官网 URL
- 域名
- 行业
- 子行业
- 公司简介
- 成立时间
- 员工规模
- 企业类型

### 4.2 联系方式与联系人信息

- 企业邮箱
- 联系电话
- WhatsApp 号码
- 联系人姓名
- 联系人职位
- 联系人邮箱
- 联系人电话
- 联系人 LinkedIn 链接

### 4.3 商业标签与营销标签

- 采购关键词标签
- 公司活跃标签
- 业务类型标签
- 市场标签
- 采购偏好标签
- 是否适合批发小单
- 是否支持低 MOQ
- 是否存在试单倾向

### 4.4 来源与审计字段

- 来源平台
- 来源页面名称
- 来源页面 URL
- 平台内部记录 ID
- 抓取时间
- 抓取批次号
- 原始截图路径或快照标识

## 9. 字段使用目标

这些字段最终服务于以下目标：

- 搜索结果展示
- 公司画像和联系人画像
- 海关活跃度评分
- 批发小单 / 大单采购偏好判断
- 后续 AI 提示词和推荐理由生成

## 10. 当前建议

建议开发上采用：

- 字段标准按增强版设计
- 抓取实施按 `P0 -> P1 -> P2` 渐进上线

这样既保证后续扩展不返工，也不至于首版采集工作量失控。
