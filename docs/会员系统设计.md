## 一、当前状态

- **无用户认证**：没有 JWT/OAuth2/Session，系统为单用户内部工具
- **无用量控制**：任何人都能无限制使用搜索、AI 评估、导出功能
- **AI 配置前端传入**：API Key 在 localStorage 明文存储，需要改为服务端管理

## 三、数据库模型

### 3.1 User（用户表）

```python
class User(Base):
    __tablename__ = "user"

    id: int                 # PK
    email: str              # 唯一，登录账号
    password_hash: str      # bcrypt 加密
    name: str               # 显示名称
    tier: str               # "free" | "pro" | "enterprise"，默认 "free"
    is_active: bool         # 是否启用
    is_admin: bool          # 管理员标记
    created_at: datetime
    updated_at: datetime
```

### 3.2 Membership（会员订阅表）

```python
class Membership(Base):
    __tablename__ = "membership"

    id: int                 # PK
    user_id: int            # FK -> user.id
    tier: str               # "free" | "pro" | "enterprise"
    started_at: datetime    # 生效时间
    expires_at: datetime    # 过期时间（null=永不过期）
    created_at: datetime
```

### 3.3 UsageLog（用量日志表）

```python
class UsageLog(Base):
    __tablename__ = "usage_log"

    id: int                 # PK
    user_id: int            # FK -> user.id
    action_type: str        # "search" | "ai_eval" | "export"
    detail: str             # JSON 详情（如搜索关键词、AI调用类型）
    period: str             # "2026-04" 格式，按月统计
    created_at: datetime
```

### 3.4 AiUsageLog（AI 调用明细表）

```python
class AiUsageLog(Base):
    __tablename__ = "ai_usage_log"

    id: int                 # PK
    user_id: int            # FK -> user.id
    call_type: str          # "evaluate" | "extract" | "summarize" | "analyze_structure"
    model: str              # 使用的模型名
    input_tokens: int       # 输入 token 数
    output_tokens: int      # 输出 token 数
    duration_ms: int        # 调用耗时
    created_at: datetime
```

## 五、技术方案

### 5.1 用户认证（后端）

- JWT 认证：`python-jose` + `passlib[bcrypt]`
- 接口：
  - `POST /api/auth/register` — 注册
  - `POST /api/auth/login` — 登录（返回 access_token）
  - `GET /api/auth/me` — 获取当前用户信息
- 依赖注入：`get_current_user` 从 `Authorization: Bearer <token>` 解析用户
- 所有业务接口加 JWT 依赖注入

### 5.2 会员管理（后端）

- 配额检查：搜索/AI评估/导出前检查当月用量
- 接口：
  - `GET /api/membership/usage` — 查询当前用量和配额
  - `POST /api/membership/upgrade` — 升级会员（暂不做支付对接，管理员手动审批）
- 管理员接口：
  - `GET /api/admin/users` — 用户列表
  - `PUT /api/admin/users/{id}/tier` — 修改用户等级
  - `GET /api/admin/usage` — 全局用量统计

### 5.3 AI 配置服务端化

- `config.py` 新增：`ai_api_key`, `ai_base_url`, `ai_model`（从 `.env` 读取）
- `AIExtractor` 改为从 `settings` 读取配置，不再从前端传入
- 搜索请求不再传 `ai_config`，后端自动使用服务端配置
- `AiUsageLog` 记录每次 AI 调用的 token 消耗

### 5.4 前端改动

- 新增页面：
  - `/login` — 登录页
  - `/register` — 注册页
- `AppShell` 改动：
  - 加用户头像/名称 + 会员状态徽章
  - 未登录显示「登录」按钮
  - 登录后显示用户名 + 退出
- `SettingsPage` 改动：
  - 新增「会员与用量」Tab：显示当前等级、用量进度条、升级按钮
  - 移除「AI 配置」Tab（改为服务端管理）
- `SearchWorkbench` 改动：
  - 搜索时配额不足 → 弹窗提示升级
  - 导出时配额不足 → 弹窗提示升级
  - 移除 `aiConfig` localStorage 读写和传递
