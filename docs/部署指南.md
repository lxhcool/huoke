# Huoke 部署指南

> 域名：https://huoke.lxhcoool.cn  
> 服务器：175.24.199.30

---

## 一、服务器前提条件

```bash
# 1. 安装 Docker
curl -fsSL https://get.docker.com | sh
systemctl start docker
systemctl enable docker

# 2. 安装 Docker Compose 插件
apt-get install docker-compose-plugin
```

---

## 二、快速部署

### 方式一：远程一键部署（推荐）

```bash
# 在本地项目根目录执行
bash scripts/remote-deploy.sh root@175.24.199.30
```

### 方式二：手动部署

```bash
# 1. 上传代码
rsync -avz --delete \
  --exclude='.git' --exclude='node_modules' --exclude='.next' \
  --exclude='__pycache__' --exclude='.venv' --exclude='*.db' \
  --exclude='.codebuddy' --exclude='.env' \
  ./ root@175.24.199.30:/opt/huoke/

# 2. SSH 到服务器
ssh root@175.24.199.30

# 3. 配置环境变量
cd /opt/huoke
cp apps/api/.env.production apps/api/.env
vim apps/api/.env
# 必填：JOINF_USERNAME, JOINF_PASSWORD, AI_API_KEY

# 4. 构建并启动
docker compose build
docker compose up -d
```

## 三、SSL 说明

SSL 已由外部处理（CDN/云服务），Nginx 容器只需监听 80 端口做反代即可。

如果需要 Nginx 自己处理 SSL：
1. 将证书文件放到 `nginx/ssl/` 目录
2. 取消 `docker-compose.yml` 中 ssl volumes 的注释
3. 取消 `nginx/nginx.conf` 中 ssl 配置的注释

## 六、常用命令

```bash
# 更新部署（代码变更后）
docker compose up -d --build

# 重启服务
docker compose restart

# 停止服务
docker compose down

# 查看日志
docker compose logs -f api
docker compose logs -f web

# 数据库操作
docker compose exec postgres psql -U huoke -d huoke

# 备份数据库
docker compose exec postgres pg_dump -U huoke huoke > backup.sql
```

## 七、故障排查

| 问题 | 排查 |
|------|------|
| API 502 | `docker compose logs api` 查看错误 |
| CORS 报错 | 检查 `.env` 中 `ALLOWED_ORIGINS` 是否包含 `https://huoke.lxhcoool.cn` |
| Playwright 失败 | `docker compose exec api playwright install chromium --with-deps` |
| 端口冲突 | `ss -tlnp \| grep -E '80\|3000\|8000'` |
